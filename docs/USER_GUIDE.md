# Sovereign Engine User Guide

This guide is for users of a Sovereign Engine instance. It covers how to use the portal, chat with AI models, manage API tokens, browse models, and request GPU reservations.

> **Note:** This guide assumes an administrator has already deployed and configured Sovereign Engine for your organisation. If you need deployment or administration help, ask your administrator for the Deployment Guide and Architecture Overview.

## Getting Started

### Logging In

Open your browser and navigate to the portal URL provided by your administrator (typically `https://your-domain/portal`).

You'll see the **Sovereign Engine** sign-in page with one or more sign-in options:

- **SSO / OIDC** — Click **"Sign in with [Provider Name]"** to authenticate through your organisation's identity provider (e.g. Google Workspace, Okta, Keycloak).
- **Bootstrap / Basic Auth** — If enabled by your administrator, click the **"Bootstrap / Basic Auth"** link to reveal a username and password form.

After signing in, you'll be taken to the Dashboard.

### Session

Your session lasts 24 hours. After that you'll be prompted to sign in again. You can sign out at any time from the user menu.

### User Menu

Click your name in the top-right corner of the navigation bar to access:

- **Theme selector** — Switch between **System**, **Light**, and **Dark** themes. Your preference is saved locally.
- **Sign Out** — End your session immediately.

## Portal Overview

The navigation bar at the top of every page provides quick access to all features:

| Link | Description |
|------|-------------|
| **Dashboard** | Your personal usage analytics |
| **Tokens** | Create and manage API access tokens |
| **Models** | Browse available models and search HuggingFace |
| **Reservations** | View the schedule and request exclusive GPU time |
| **Chat** | Opens the chat interface (Open WebUI) |

On the right side of the navigation bar you'll also see:

- **GPU Status** — A live indicator showing current GPU memory usage and utilisation percentage for each GPU. The bar turns green (≤70%), amber (>70%), or red (>90%) based on load. Hover over it for details.
- **Your name** — Opens the user menu (theme and sign out).

> If the system is currently reserved by another user, a warning banner appears below the navigation bar: *"System is currently reserved for exclusive use by [name]"*.

## Chat Interface

The **Chat** link in the navigation bar takes you to Open WebUI — a full-featured chat interface for conversing with AI models.

### Getting Started with Chat

1. Click **Chat** in the navigation bar.
2. You're automatically signed in via SSO — no separate login is needed.
3. Select a model from the available options.
4. Type your message and press Enter to start a conversation.

### Features

- **Model selection** — Choose from any model currently loaded on the system.
- **Conversations** — Your chat history is saved automatically. You can return to previous conversations at any time.
- **File uploads** — Attach documents and files to your messages for the model to analyse.
- **Multiple conversations** — Start new conversations while keeping previous ones accessible.

> **Tip:** All chat activity counts toward your personal usage metrics, visible on the Dashboard.

## Dashboard

The Dashboard (`/portal/`) shows your personal API usage analytics.

### Summary Cards

Three cards at the top display totals for the selected time period:

- **Total Requests** — Number of API calls you've made.
- **Input Tokens** — Total tokens sent to models.
- **Output Tokens** — Total tokens generated by models.

### Time Period

Use the period selector in the top-right corner to choose a time window:

- **Hour** — Last 60 minutes
- **Day** — Last 24 hours (default)
- **Week** — Last 7 days
- **Month** — Last 30 days

### Charts

The dashboard displays six charts arranged in three rows:

**Row 1 — Model Usage**
- **Model Usage** (line chart) — Requests over time, broken down by model.
- **Requests by Model** (pie chart) — Proportion of requests per model.

**Row 2 — Token Usage**
- **Access Token Usage** (line chart) — Requests over time, broken down by API token.
- **Requests by Token** (pie chart) — Proportion of requests per API token.

**Row 3 — Token Consumption**
- **Token Usage** (line chart) — Input and output token counts over time.
- **Token Split** (pie chart) — Ratio of input to output tokens.

All charts refresh automatically every 15 seconds.

## API Tokens

The Tokens page (`/portal/tokens`) lets you create and manage API tokens for programmatic access to Sovereign Engine's OpenAI-compatible API.

### Creating a Token

1. Click **"Create Token"** in the top-right corner.
2. Fill in the form:
   - **Token Name** (required) — A descriptive name, e.g. `my-project-key`.
   - **Category** — Optionally restrict the token to a specific model category (e.g. "thinking", "coding"). Leave as *"No category restriction"* for access to all models.
   - **Expires In** — Choose an expiry period: 30, 60, 90 (default), or 180 days, or 1 year.
   - **Advanced Options** — Click *"Show Advanced Options"* to reveal:
     - **Specific Model** — Restrict the token to a single model. This overrides category routing.
3. Click **"Create Token"**.

### Saving Your Token

After creation, your token is displayed **once** in a warning panel:

> ⚠️ **Save this token now — it will not be shown again.**

- Click **"Copy"** to copy the token to your clipboard.
- Store it somewhere secure (e.g. a password manager or environment variable).
- The token format is `se-XXXXXXXX-XXXX-...`.

If you lose the token, you'll need to create a new one.

### Managing Tokens

Your tokens are listed in a table showing:

| Column | Description |
|--------|-------------|
| **Name** | The name you gave the token |
| **Category** | Category restriction, if any |
| **Expires** | Expiry date, or *"Never"* |
| **Created** | When the token was created |
| **Status** | **Active** (green), **Expired** (amber), or **Revoked** (red) |

**Actions:**

- **Revoke** — Immediately disables the token. This cannot be undone. Use this if a token is compromised.
- **Delete** — Permanently removes the token from your list. If the token was still active, it stops working immediately.

## Models

The Models page (`/portal/models`) lets you browse available models and download new ones from HuggingFace.

### Registered Models

A table lists all models registered on the system:

| Column | Description |
|--------|-------------|
| **Repository** | The HuggingFace repository name |
| **File** | The specific model file |
| **Size** | File size on disk |
| **Context** | Maximum context length (tokens) |
| **Backend** | Inference backend (e.g. llama.cpp) |
| **Status** | **Loaded** (green) = ready to use, **Not Loaded** (grey) = available but not running |

> Only **Loaded** models can be used for chat and API requests.

### Searching HuggingFace

To find and download new models:

1. Enter a search term in the **"Search models..."** field.
2. Optionally select a task type from the dropdown (e.g. *text-generation*, *summarization*).
3. Click **"Search"** or press Enter.
4. Browse results showing model name, task, download count, and likes.
5. Click **"Download"** on any model to start downloading it.

For **GGUF models**, clicking Download opens a file picker where you can select the specific `.gguf` file to download.

### Active Downloads

When downloads are in progress, an **"Active Downloads"** section appears showing:

- Repository name and download progress (percentage bar)
- Downloaded and total file size
- A **"Cancel"** button to stop the download

### Disk Usage

A disk usage bar at the top of the page shows how much storage space is used:

- **Green** — Under 80% used
- **Amber** — 80–95% used
- **Red** — Over 95% used (downloads are blocked)

## GPU Reservations

The Reservations page (`/portal/reservations`) lets you request exclusive access to the system's GPU resources for a specific time window.

### Why Reservations?

Outside of reservations, Sovereign Engine uses **fair-use scheduling** — requests from all users are queued fairly to prevent any one person from monopolising the system. Reservations give you exclusive access when you need guaranteed, uncontested GPU time.

### Viewing the Schedule

The **weekly calendar** shows all reservations colour-coded:

- **Blue** — Your reservations
- **Green** — Other users' reservations
- **Orange** — Currently active reservation
- **Grey (semi-transparent)** — Pending (not yet approved)

A red line marks the current time. Use the **Prev**, **This Week**, and **Next** buttons to navigate between weeks. Toggle between **07:00–22:00** and **24h** views.

Hover over any reservation block to see the user, status, time range, and reason.

### Requesting a Reservation

1. **Click and drag** across empty time slots on the calendar to select your desired window.
2. A dialog appears with:
   - **Start** and **End** times (pre-filled from your selection, adjustable).
   - **Reason** (optional) — Describe what you need the system for.
3. Click **"Request Booking"**.

Your reservation will appear on the calendar as **Pending** until an administrator approves it.

### Viewing Your Reservations

Click **"My Reservations"** (top-right of the schedule section) to see all your reservations with their status:

| Status | Meaning |
|--------|---------|
| **Pending** | Awaiting admin approval |
| **Approved** | Approved, waiting for the scheduled start time |
| **Active** | Currently in progress — you have exclusive access |
| **Completed** | Finished normally |
| **Rejected** | Declined by an administrator (check the Admin Note for details) |
| **Cancelled** | Cancelled by you |

You can **Cancel** any reservation that is still Pending or Approved.

### During an Active Reservation

When your reservation is active, a green panel appears at the top of the Reservations page:

> **"You hold the active reservation"**
> You can start and stop model containers during your reservation.

You'll see a list of all registered models with **"Start"** and **"Stop"** buttons. Use these to load models onto the GPU for your exclusive use during your reservation window.

Other users will see a banner informing them the system is reserved.

## Using the API Programmatically

Sovereign Engine provides an **OpenAI-compatible API** at `/v1/chat/completions` and an **Anthropic-compatible API** at `/v1/messages`. Any tool or library that works with either API can be pointed at Sovereign Engine instead.

### Quick Start

1. [Create an API token](#creating-a-token) from the Tokens page.
2. Use the token in your API requests as a Bearer token or `x-api-key` header.

### Example: curl

```bash
curl https://your-domain/v1/chat/completions \
  -H "Authorization: Bearer se-your-token-here" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-model-name",
    "messages": [
      {"role": "user", "content": "Hello, how are you?"}
    ]
  }'
```

### Example: Python (OpenAI SDK)

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://your-domain/v1",
    api_key="se-your-token-here"
)

response = client.chat.completions.create(
    model="your-model-name",
    messages=[
        {"role": "user", "content": "Hello, how are you?"}
    ]
)

print(response.choices[0].message.content)
```

### Example: Node.js (OpenAI SDK)

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'https://your-domain/v1',
  apiKey: 'se-your-token-here',
});

const response = await client.chat.completions.create({
  model: 'your-model-name',
  messages: [
    { role: 'user', content: 'Hello, how are you?' }
  ],
});

console.log(response.choices[0].message.content);
```

### Example: Python (Anthropic SDK)

Sovereign Engine also supports the Anthropic Messages API at `/v1/messages`. You can use the official Anthropic Python SDK by pointing it at your instance:

```python
import anthropic

client = anthropic.Anthropic(
    base_url="https://your-domain/v1",
    api_key="se-your-token-here",
)

message = client.messages.create(
    model="your-model-name",
    max_tokens=256,
    messages=[
        {"role": "user", "content": "Hello, how are you?"}
    ],
)

print(message.content[0].text)
```

Streaming is also supported:

```python
with client.messages.stream(
    model="your-model-name",
    max_tokens=256,
    messages=[{"role": "user", "content": "Tell me a story."}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
```

### Example: Node.js (Anthropic SDK)

```javascript
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic({
  baseURL: 'https://your-domain/v1',
  apiKey: 'se-your-token-here',
});

const message = await client.messages.create({
  model: 'your-model-name',
  max_tokens: 256,
  messages: [
    { role: 'user', content: 'Hello, how are you?' }
  ],
});

console.log(message.content[0].text);
```

> **Note:** The Anthropic SDK sends your token via the `x-api-key` header by default. Sovereign Engine accepts both `x-api-key` and `Authorization: Bearer` headers, so no extra configuration is needed.

### Example: Claude Code

You can point [Claude Code](https://docs.anthropic.com/en/docs/claude-code) at your Sovereign Engine instance to use it as the AI backend:

```bash
ANTHROPIC_BASE_URL=https://your-domain ANTHROPIC_AUTH_TOKEN=se-your-token-here \
  claude --model your-model-name -p "Your prompt here"
```

For example, to run a one-shot prompt:

```bash
ANTHROPIC_BASE_URL=https://your-domain ANTHROPIC_AUTH_TOKEN=se-your-token-here \
  claude --model your-model-name --dangerously-skip-permissions \
  -p "Please create a hello world python script called hello.py"
```

You can also export these as environment variables in your shell profile so Claude Code always uses your Sovereign Engine instance:

```bash
export ANTHROPIC_BASE_URL=https://your-domain
export ANTHROPIC_AUTH_TOKEN=se-your-token-here
```

### Available Endpoints

Key endpoints:

| Endpoint | Description |
|----------|-------------|
| `GET /v1/models` | List available models |
| `POST /v1/chat/completions` | Chat completion (OpenAI format) |
| `POST /v1/messages` | Chat completion (Anthropic format) |

For the complete API specification, ask your administrator for the API Reference document.

If your token is restricted to a specific **category** or **model**, requests outside that scope will be rejected. You can check your token's restrictions on the Tokens page.

## Tips & Troubleshooting

### General Tips

- **Check model status** — If a model shows as *"Not Loaded"* on the Models page, it's not available for chat or API requests. Ask your administrator to load it, or load it yourself during a GPU reservation.
- **Watch GPU usage** — The GPU status indicator in the navigation bar gives you a quick sense of system load. High utilisation may mean slower responses.
- **Use categories** — When creating API tokens, category restrictions ensure your requests are routed to the most appropriate model for your use case.

### Common Issues

**"No models available" in chat**
- No models are currently loaded. Check the Models page — all models may show as *"Not Loaded"*.
- Contact your administrator to load a model.

**API requests return 401 Unauthorized**
- Your token may be expired or revoked. Check the Tokens page for its status.
- Ensure you're including the `Authorization: Bearer se-...` header.

**API requests return 503 or timeout**
- The system may be under heavy load. The fair-use scheduler queues requests — try again shortly.
- If the system is reserved by another user, non-reservation requests may be deprioritised.

**Can't create a reservation**
- Ensure the time slot you're selecting is in the future and doesn't overlap with an existing reservation.

**Reservation was rejected**
- Check the **Admin Note** in your My Reservations dialog for the reason.
- Try requesting a different time slot.

**Chat interface requires separate login**
- This shouldn't happen — chat uses the same session as the portal. Try signing out and back in.
- Clear your browser cookies for the domain and sign in again.
